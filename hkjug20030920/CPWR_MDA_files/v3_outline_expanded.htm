What is MDA?

Model-Driven Architecture
Object Management Group
A new way to specify and build systems
Based on modeling with UML
Modeling instead of programming
Merging model and coding
Based on standards
UML, MOF, CWM, XMI…..
“MDA is about moving to the NEXT level – instead of offering another set of APIs, it offers an opportunity to model those APIs generically – and then generate code for accessing the APIs you choose (or get stuck with). More importantly from the view of the app developer, you model *your application* generically, ignoring the plethora of APIs”
                                                                        Richard Soley

Model-Driven Architecture
Clear distinction between the models and the implementation
Platform Independent Model (PIM)
Platform Specific Model (PSM)
Code Model
Business and technology changes
Constantly changing environment of today’s market
Business model (PIM) remains stable

MDA Benefits
Increase Productivity
Raising the Level of Abstraction
Artifacts of different steps in lifecycle
Abstraction of system
Shifts developer to analysis & design
More resources to solve business issues
Automation of transformation
PIM developers need not concern the platform specific details
PSM developers write less code
Doc are no longer just paper!

MDA Benefits
Enhance Portability
PIM is independent of technologies
Automatic transformed into multiple platforms (PSM) à Portable
Portability depends on availability of transformation tools
Provide concrete languages to support custom transformation definitions

MDA Benefits
Higher Interoperability
PIM transformed into multiple PSMs, each targeting a particular platform
The tool knows the relationships between each PSM
Deduces how to bridge them together
e.g. EJB à DBMS Table

MDA Benefits
Easier to Maintenance and Documentation
Developers focus on PIM
A higher abstraction than code
Models àhigh-level documentation
Change made to PIM à propagates to PSM & Code
Keep developers maintain the “doc”
Work in a more declarative way
Remains consistent with the code
Documentation becomes useful !
