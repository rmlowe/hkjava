The Portability Problem
l Software industry release new stuffs each year
– Java, Linux, XML, HTML, SOAP, UML, J2EE, .Net, Web
Service, etc…
l We need to follow because:
– Demanded by customers (Web UI)
– Solves some real problem (Web Service for integration)
– Become legacy (stop supported)
l Investment lose value or even worthless
l Technology changes no guarantee backwards
compatible
l Result àPort existing system to new platform(s)
