n Native XML DB products are sexy, but ...
u Fall apart when interrelated documents are stored
u Are not yet optimized for large amounts of data
compared to RDBs
u Only provide a single (hierarchical) view of the data
u Only return the documents that you put in
u Do not let you stitch together data from multiple
