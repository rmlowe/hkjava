The best way to understand rich internet applications is to put them in the context of other Internet Solutions. If you think about these solutions in terms of client and server functionality there are basically four major categories of solutions.

Click 1 – So the first category is websites which are basically static HTML. There are 4 billion static HTML pages on the Internet today, and these aren’t going away. In fact basic websites continue to be the foundation of information delivery on the Internet.

Click 2 – Starting back in 1995 with the release of shockwave followed by Java and Flash, browsers have been extended to support rich content that offers greater interactivity and functionality. Rich content continues to build momentum as a solution for branding, marketing, elearning, and presenting complex visual information.

Click 3 – At the same time the web has quickly been adopted as a deployment platform for applications. Starting first with the introduction of CGI, then ColdFusion in 1995 followed by ASP and other application server technologies the web has become a deliver mechanism for dynamic data driven applications and content management systems.

Click 4 – Rich Internet Applications are the next logical evolution. They bring together the rich interactive capabilities of rich clients with the dynamic application capabilities of server scripting environments and application servers.
