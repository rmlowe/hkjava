Objective:
•Communicate the flexibility of the Flash Remoting Service
•Make it clear that purchasing ColdFusion MX also gives you access to Flash Remoting for .NET and Java

As with other functionality within ColdFusion, we provide multiple ways to take advantage of the Flash Remoting Service. As I mentioned before, you can expose any method on a component to the Flash Remoting service. You can also write variables from individual ColdFusion pages using the Flash.Results variable.

We’ve also introduced support for a new language in the ColdFusion scripting environment. With ColdFusion MX, you can define server-side functions in ActionScript. This enables a Flash developer to use their familiar scripting language to functions that are performed on the server, such as querying a database or interacting with other server resources, without having to learn an addition language. Thus, if a Flash developer wants to build the entire application in ActionScript, they can do that. Or if the team is split among server-side ColdFusion developers and UI developers working in Flash, they can easily work together – with the Flash developers consuming reusable components built by the ColdFusion developers.

Lastly, ColdFusion MX also ships with the components for using the Flash Remoting Service with .NET and Java objects, offering you the ultimate flexibility in building server-side accessible from Macromedia Flash. These components must be installed separately.
